📁 Package storage

internal/storage/

S3-compatible object storage client used for avatar image uploads. Built on AWS SDK v2, supporting any S3-compatible backend — AWS S3, Cloudflare R2, DigitalOcean Spaces, or MinIO in local development.

Files

FileResponsibility
s3.goS3Client struct with upload, delete, and public URL generation methods.

S3Client

Constructor

func New(cfg *config.Config) (*S3Client, error)

Initialises the AWS SDK v2 client with static credentials from config. If a custom S3Endpoint is set, the client uses path-style addressing for compatibility with MinIO and other self-hosted providers.

Methods

  • PutObject(ctx, key, contentType string, r io.Reader) error Streams an object to the configured S3 bucket. The key is the object path within the bucket (e.g. avatars/<userID>.webp). The object is stored with the given content type and public-read ACL.
  • DeleteObject(ctx context.Context, key string) error Removes an object from the bucket by key. Called when a user uploads a new avatar to clean up the old one, and when an account is deleted.
  • PublicURL(key string) string Returns the public HTTPS URL for an object. Format depends on the provider — for AWS S3 it's https://<bucket>.s3.<region>.amazonaws.com/<key>; for custom endpoints it uses the endpoint URL directly.

Avatar Upload Flow

// In UsersHandler.UploadAvatar:
1. Validate Content-Type is image/webp, image/jpeg, or image/png
2. Limit request body to 2MB with http.MaxBytesReader
3. Generate object key: "avatars/" + userID + extension
4. s3.PutObject(ctx, key, contentType, r.Body)
5. publicURL := s3.PublicURL(key)
6. userStore.UpdateProfile(ctx, userID, "", publicURL)
7. Delete old avatar key if one existed

Local Development

In development, the S3 fields in .env can point to a MinIO container. Add the following to docker-compose.yml:

minio:
  image: minio/minio
  command: server /data --console-address ":9001"
  environment:
    MINIO_ROOT_USER: minioadmin
    MINIO_ROOT_PASSWORD: minioadmin
  ports:
    - "9000:9000"
    - "9001:9001"

Then set S3_ENDPOINT=http://minio:9000 in your .env. If no S3 config is present, avatar upload will fail gracefully with a 503 and a descriptive error rather than panicking.

Phase 3 feature: S3 storage is gated behind Phase 3. The storage package exists and is wired in server.New(), but the avatar upload route may return a 503 if S3 credentials are not configured.
