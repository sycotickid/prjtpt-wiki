⚙️ Package config

internal/config/

Single source of truth for all runtime configuration. Reads environment variables (with .env file fallback in development), validates required fields, and exposes a typed Config struct to the rest of the application.

Config Struct

All fields on the Config struct are populated from environment variables. Required fields cause Load() to return an error if missing.

Load() Function

config.Load() is called once at startup in main(). It:

  1. Calls godotenv.Load() to source a .env file if present (silently ignored in production).
  2. Reads each environment variable with os.Getenv.
  3. Returns an error listing every missing required variable so operators see all problems at once rather than fixing them one at a time.

Helper Methods

  • IsDevelopment() bool Returns true when APP_ENV == "development". Used to enable hot-reload CSP, verbose logging, and .env loading.
  • IsProduction() bool Returns true when APP_ENV == "production". Enables HSTS, strict CSP, and JSON-only logging.

Never commit secrets. The .env file is in .gitignore. Use .env.example as a template — it contains placeholder values and descriptions for every variable but no real secrets.
