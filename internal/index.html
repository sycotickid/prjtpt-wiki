📦 Private Packages

internal/

The internal/ directory contains all private application packages. Go enforces that code inside internal/ cannot be imported by packages outside the module root — this is not a convention but a compiler rule. All business logic, data access, and HTTP handling lives here.

Design Principles

Dependency Injection via Constructor

Every package exposes a constructor function (e.g. auth.NewJWTService, store.NewUserStore) that accepts its dependencies explicitly. There is no global state or init() magic. server.New() acts as the composition root, wiring everything together.

Context Propagation

All store methods accept context.Context as their first argument, enabling request-scoped cancellation and timeout propagation from the HTTP layer down to database queries.

Parameterized Queries

Every database query uses pgx/v5 parameterized arguments — no string interpolation, no fmt.Sprintf in SQL. This eliminates SQL injection at the architecture level rather than relying on developer discipline.

Domain Layer (store/)

Store structs own their database queries. Handlers call stores; stores do not call handlers. This one-way dependency keeps the data access layer independently testable.
