📜 Package logger

internal/logger/

Configures a zerolog.Logger instance tuned for the current environment. Development gets human-readable coloured output; staging and production get machine-parseable JSON at a higher log level.

Files

FileResponsibility
logger.goConstructs and returns a configured zerolog.Logger.
logger_test.goTests that verify the correct log level and writer are used per environment.

Functions

  • New(cfg *config.Config) zerolog.Logger Returns a logger configured for the current environment. Passes os.Stderr as the writer.
  • NewWithWriter(cfg *config.Config, w io.Writer) zerolog.Logger Like New but accepts a custom writer — used in tests to capture log output for assertions without writing to stderr.

Environment Behaviour

APP_ENVLevelFormat
developmentDEBUGPretty-printed with colour, timestamps, and readable field names (zerolog ConsoleWriter)

productionINFOJSON to stderr — suitable for log aggregators

Log Fields

Every log entry includes a service field set to "prjtpt-api" and a time field in RFC3339 format. The request logging middleware (see middleware/logger.go) adds method, path, status, latency_ms, ip, and request_id to each HTTP log line.

Example JSON log line (production)

{
  "level":"info",
  "service":"prjtpt-api",
  "time":"2026-04-02T14:23:01Z",
  "method":"POST",
  "path":"/api/v1/auth/login",
  "status":200,
  "latency_ms":42,
  "ip":"203.0.113.5",
  "request_id":"abc123"
}

Why zerolog? zerolog is the fastest structured logger available in the Go ecosystem — its zero-allocation design means logging has negligible impact on request latency even at high throughput. JSON output integrates directly with log aggregators like Loki, Datadog, or CloudWatch without a parser plugin.
