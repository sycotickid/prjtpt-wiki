📷 Package presence

internal/presence/

Implements a WebSocket-based real-time presence hub. Connected clients receive live events about their friends' status changes, friend requests, lobby invites, and lobby activity. The hub is the central nervous system for all real-time social features.

Files

FileResponsibility
hub.goHub and Client structs — connection lifecycle, event broadcasting, offline grace period.

Architecture

Client A (WebSocket) ──► Hub ──► broadcast to Client B, C, D
                          │
                          ├── clientsByUserID map[string][]*Client
                          ├── register / unregister channels
                          └── offline grace timers (30s)

The Hub runs in a single goroutine, processing registrations, unregistrations, and broadcast requests via channels. This single-writer pattern means no mutex is needed on the client maps.

Each Client runs two goroutines: a reader (listens for ping/pong frames and client disconnect) and a writer (drains an outbound message queue). The client is unregistered when either goroutine exits.

Hub

Constructor

func NewHub(friendStore FriendStore, log zerolog.Logger) *Hub

Methods

  • Run() Starts the hub event loop in a goroutine. Must be called once at server startup. Processes register, unregister, and broadcast channel messages.
  • Register(client *Client) Adds a client to the hub and cancels any pending offline grace timer for the user. Emits a status_change → online event to the user's friends.
  • Unregister(client *Client) Starts the 30-second offline grace period. If no other connection from the same user registers within that window, the user is marked offline and friends are notified.
  • BroadcastToUser(userID string, event PresenceEvent) Sends an event to all WebSocket connections belonging to a specific user (they may have multiple tabs open).
  • BroadcastToFriends(userID string, event PresenceEvent) Looks up the user's friend list, then calls BroadcastToUser for each friend that is currently connected.

PresenceEvent

The JSON payload sent over every WebSocket connection. The type field determines which other fields are populated.

Offline Grace Period

When a client disconnects (e.g. page refresh, network blip), a 30-second timer starts before friends are notified that the user went offline. If the user reconnects within that window, the timer is cancelled and no offline event is emitted. This prevents notification spam from brief disconnections.

WebSocket Endpoint

Clients connect to GET /ws/presence?token=<JWT>. The JWT is validated before the WebSocket upgrade is performed. The gorilla/websocket library handles the HTTP → WebSocket protocol upgrade, ping/pong keepalive, and graceful close frames.
