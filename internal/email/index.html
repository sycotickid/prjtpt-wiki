💌 Package email

internal/email/

Provides a lightweight transactional email mailer. In production it delivers messages via SMTP. In development (no SMTP_HOST configured) it prints formatted emails to stdout so developers can see the content without a real mail server.

Mailer

Constructor

func New(cfg *config.Config) *Mailer

Reads SMTP credentials from config.Config. If SMTPHost is empty the mailer runs in development mode and logs to stdout instead of connecting to a mail server.

Methods

  • SendVerification(addr, displayName, verifyURL string) error Sends a "Verify your email" message to addr. The body contains a link to verifyURL which the user clicks to confirm ownership. The link embeds the raw token generated by auth.GenerateToken().
  • SendPasswordReset(addr, displayName, resetURL string) error Sends a "Reset your password" message containing a time-limited link. The link expires in 1 hour (enforced by the user store when checking the token).

Message Format

Emails are formatted as plain RFC 2822 messages (headers + CRLF body). HTML-only emails are avoided to maximise deliverability and avoid spam filters. Future phases may add multipart MIME with both plain-text and HTML parts.

Development Mode

When no SMTP_HOST is configured, the mailer writes the email content to the logger at info level. This means during local development you can see exactly what would be emailed without needing an actual SMTP server like Mailhog or Mailtrap.

Testing emails locally: Either leave SMTP unconfigured to see output in the server log, or add a Mailhog container to docker-compose.yml and point SMTP_HOST=mailhog at it to capture emails in a local web UI.

SMTP Authentication

Uses smtp.PlainAuth (username/password) over a TLS-upgraded connection. For production deployments, a transactional email provider (Postmark, SendGrid, AWS SES) is recommended over a raw SMTP relay.
