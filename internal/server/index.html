internal/server/

The composition root of the application. server.New() builds the entire dependency graph — database pool, stores, services, handlers — and wires them onto the chi router with the full middleware stack.

Files

server.go — Composition Root

New() — Assembly Order

server.New(cfg, log) performs dependency injection in this order:

  1. Open PostgreSQL pool via openDB(cfg)
  2. Instantiate all store implementations (UserStore, FriendStore, GameStore, LobbyStore, MessageStore, PushSubscriptionStore, RefreshTokenStore)
  3. Construct service instances: auth.JWTService, storage.S3Client, email.Mailer, presence.Hub
  4. Start the presence hub goroutine (hub.Run())
  5. Instantiate all handler structs, injecting their store/service dependencies
  6. Build the chi router, attach global middleware, register all route groups
  7. Return the configured *http.Server

db.go — Database Pool

openDB()

Creates a pgxpool.Pool using the DATABASE_URL from config. The pool is configured with:

SettingValueReason
MinConns5Keep warm connections ready
MaxConns25Match PostgreSQL's default max_connections allocation
MaxConnLifetime1 hourPrevent stale connections from accumulating
MaxConnIdleTime30 minRelease unused connections to PostgreSQL
HealthCheckPeriod1 minDetect dead connections before they're used

The function retries the connection up to 5 times with exponential backoff, accommodating Docker Compose startup ordering where the API container may start before PostgreSQL is fully ready.
