🌐 Package handler

internal/handler/

All HTTP request handlers. Each domain area has its own file containing a handler struct whose methods map 1:1 to route endpoints. Handlers decode requests, call the store or service layer, and encode responses — they contain no business logic of their own.

auth.go — AuthHandler

Manages the full authentication lifecycle. Tokens are issued by the auth package; user records are persisted via UserStore and RefreshTokenStore.

POST/auth/registerValidates email/password/displayName, bcrypt-hashes the password, creates the user, issues a verification token, sends a verification email.
POST/auth/loginVerifies credentials, checks email is verified, issues a 15-min JWT access token and sets an HttpOnly refresh token cookie.
GET/auth/verify-email?token=...Hashes the presented token, looks it up in the DB, marks the user as verified.
POST/auth/resend-verificationIssues a new verification token and re-sends the email (rate-limited).
POST/auth/forgot-passwordFinds user by email, generates a password-reset token (1-hour TTL), sends reset email. Always returns 200 to avoid user enumeration.
POST/auth/reset-passwordValidates the reset token, hashes the new password, updates the user, clears the token.
POST/auth/refreshReads the refresh cookie, validates and rotates the token (issues a new one, revokes the old), returns a new access token. Detects token reuse (family revocation).
DELETE/auth/sessionRevokes the current refresh token family, clears the cookie.

users.go — UsersHandler

GET/api/v1/users/meReturns the authenticated user's profile (ID, email, displayName, avatarURL, status, notificationsEnabled).
PATCH/api/v1/users/meUpdates displayName. Validates uniqueness against the DB before saving.
POST/api/v1/users/me/avatarAccepts multipart image upload (max 2MB, JPEG/PNG/WebP). Streams to S3, updates avatarURL in DB, deletes old object.
PATCH/api/v1/users/me/statusUpdates status (online/offline/away/in_game/in_lobby/custom) and optional customStatus string. Broadcasts a status_change event to friends via the presence hub.
GET/api/v1/users/search?q=...Trigram-similarity search over display names. Returns up to 20 matches. Excludes blocked users.

friends.go — FriendsHandler

GET/api/v1/friendsReturns accepted friends with their live online status (from presence hub).
GET/api/v1/friends/requestsReturns incoming pending friend requests.
POST/api/v1/friends/requestsCreates a pending friendship record. Notifies the recipient via presence hub (friend_request event).
PATCH/api/v1/friends/requests/{id}Accepts or declines a request. Notifies the requester via presence hub (friend_accepted or friend_declined).
DELETE/api/v1/friends/{id}Removes an accepted friendship. Notifies via presence hub (friend_removed).
POST/api/v1/friends/{id}/blockBlocks a user — they can no longer send requests or find you in search.

games.go — GamesHandler

GET/api/v1/gamesReturns all active games from the catalogue (seeded by migration 000011).
GET/api/v1/games/{slug}Returns a single game by URL slug (e.g. /api/v1/games/typing-speed).

lobbies.go — LobbiesHandler

POST/api/v1/lobbiesCreates a lobby for a specific game. The creator is automatically added as the host/first member.
GET/api/v1/lobbiesLists all public lobbies with status "waiting".
GET/api/v1/lobbies/{id}Returns a lobby with its full member list.
DELETE/api/v1/lobbies/{id}Soft-deletes the lobby (sets closed_at). Only the host can do this.
POST/api/v1/lobbies/{id}/joinAdds the authenticated user to the lobby (if under max_players and status is "waiting").
DELETE/api/v1/lobbies/{id}/members/meRemoves the user from the lobby. If the host leaves, a new host is assigned.
PATCH/api/v1/lobbies/{id}/members/meToggles the user's is_ready flag. Emits lobby_all_ready event if all members are ready.
POST/api/v1/lobbies/{id}/invitesSends a lobby invite to a friend via the presence hub (lobby_invite event).
POST/api/v1/lobbies/{id}/startHost-only. Changes lobby status to in_game, creates a game_session record, emits lobby_game_starting event.
POST/api/v1/lobbies/{id}/messagesPosts a chat message to the lobby channel. Broadcasts to all connected lobby members.
GET/api/v1/lobbies/{id}/messagesReturns all messages in the lobby's chat channel, joined with sender profile.

push.go — PushHandler

GET/api/v1/push/vapid-public-keyReturns the VAPID public key so the browser can subscribe to push notifications.
POST/api/v1/push/subscribeUpserts a push subscription (endpoint + p256dh + auth) for the authenticated user.
DELETE/api/v1/push/subscribeRemoves a push subscription by endpoint.

pages.go — PagesHandler

Renders server-side HTML pages using Go's html/template engine via the renderer.go helper. All pages use the base layout in web/templates/index.html.

GET/Landing page with GitHub commit feed and product showcase.
GET/auth/signinSign-in form page.
GET/auth/registerRegistration form page.
GET/auth/verify-pendingTells the user to check their email after registration.
GET/auth/forgot-passwordForgot password form page.
GET/auth/reset-passwordReset password form page (reads token from query string).
GET/contactContact form page.
GET/productsGames catalogue / "solutions" page.
GET/lobby/{id}Pre-game lobby room page (Datastar-driven, live updates via WebSocket).
GET*Catch-all: serves the Datastar SPA shell for client-side routes not explicitly handled above.

ws.go — WSHandler

WS/ws/presence?token=<JWT>Validates the JWT query parameter, upgrades to WebSocket, registers the client with the presence hub. The connection stays open until the client disconnects or the server shuts down.

contact.go — ContactHandler

POST/contactRate-limited (5 submissions per IP per hour). Accepts name, email, message. Sends to the configured SMTP recipient. Returns a Datastar SSE fragment to update the form UI without a page reload.

Helper Files

renderer.go

Parses all HTML templates once at server startup and caches them. For each request, it clones the base layout and executes the page-specific template into it. Cloning ensures each request gets its own template namespace — concurrent rendering is safe without a mutex.

respond.go

Helper functions used by all handlers to write consistent JSON responses:

  • jsonOK(w, v)200 OK with JSON-encoded v
  • jsonCreated(w, v)201 Created with JSON-encoded v
  • jsonError(w, status, msg)Error envelope: {"error": "msg"}
  • jsonValidationError(w, fields)422 with per-field validation errors: {"error": "validation failed", "fields": {...}}
