🌐 Frontend Assets

web/

Server-rendered HTML templates and static assets. The frontend uses the Datastar hypermedia framework — pages are rendered by the Go server and made reactive through Datastar's SSE-driven signals and fragments rather than a heavyweight client-side SPA framework.

Directory Structure

PathPurpose

web/static/css/Application stylesheet
web/static/js/Datastar framework, app logic, and service worker
web/static/fonts/Self-hosted Material Icons font (offline / firewall-safe)
web/static/images/Logos, favicons, OG images

web/templates/

Go's html/template package is used for all server-rendered pages. Templates are parsed once at server startup by handler/renderer.go and cloned per request.

index.html — Base Layout

The root layout shell that every page is composed into. Contains:

  • Theme detection script: Reads prefers-color-scheme and sets a data-theme attribute before first paint to prevent flash of wrong theme.
  • CSS link: /static/css/app.css
  • Material Icons: Self-hosted via /static/fonts/material-icons.css
  • Datastar script: /static/js/datastar.js — the reactive hypermedia engine
  • Service Worker registration: Registers /static/js/sw.js for push notification support (Phase 2)
  • Navigation bar: Logo + links (Home, Solutions, Contact). The nav is part of the base layout so it appears on every page.
  • Template slot: {{block "content" .}} where page-specific content is injected

web/static/css/app.css

Single application stylesheet with no build step required. Uses CSS custom properties (design tokens) for theming.

  • Design tokens: Colour palette, spacing scale, typography sizes — defined as CSS variables on :root and overridden in [data-theme="dark"].
  • Automatic dark mode: Base tokens are dark-mode values. Light mode tokens are applied when data-theme="light" is set (via the theme detection script in the layout).
  • Component styles: Buttons, inputs, cards, navigation, modals, toasts — all themed via the token system.
  • Utility classes: Minimal set of flexbox/grid utilities. No Tailwind dependency.

web/static/js/

FilePurpose
datastar.jsSelf-hosted Datastar framework (bundled, no CDN dependency). Provides reactive signals, SSE fragment merging, and data-star-* attribute directives.
app.jsApplication-level JavaScript: WebSocket presence client, JWT token management (stores access token in memory, refreshes automatically), CSRF token header injection, push notification subscription, client-side utilities.
sw.jsService Worker: handles push notification events (push listener), shows browser notifications, and manages notification click actions (navigate to lobby/friend request).

app.js — Key Responsibilities

  • Token management: Access tokens are stored in memory (not localStorage) to prevent XSS theft. A refresh timer automatically hits POST /auth/refresh 30 seconds before expiry.
  • CSRF header injection: Intercepts all fetch calls via a global wrapper, reads the csrf_token cookie, and attaches it as X-CSRF-Token.
  • Presence WebSocket: Connects to /ws/presence?token=<JWT>, dispatches incoming events to Datastar signals, and handles reconnection with exponential backoff.
  • Push subscription: Requests notification permission, calls the VAPID public key endpoint, creates a PushSubscription, and POSTs it to /api/v1/push/subscribe.

sw.js — Service Worker

Registered on page load. Handles the browser's push event by parsing the payload and calling showNotification(). Handles notificationclick to navigate the user to the relevant page (e.g. /lobby/{id} for a lobby invite).

web/static/fonts/

Contains a self-hosted copy of the Material Symbols / Material Icons font files and their CSS declaration (material-icons.css). Self-hosting avoids external network requests, which is essential for the corporate-firewall use case where Google Fonts may be blocked.

Datastar Framework

Datastar is a hypermedia-first reactive framework that extends HTML with data-star-* attributes. Instead of shipping a large JavaScript bundle and managing client-side state, the server streams HTML fragments and JSON signals over SSE that Datastar merges into the DOM.

Key Concepts

ConceptDescription
data-signalsDeclares reactive signal values on a DOM element. Signals are like reactive variables that Datastar tracks.
data-on-clickAttaches a click handler that can trigger SSE requests or signal mutations.
data-textBinds an element's text content to a signal value — updates automatically when the signal changes.
data-showConditionally shows/hides an element based on a signal expression.
SSE mergeServer sends event: datastar-merge-fragments with HTML — Datastar patches the DOM without a full reload.

Why Datastar? It delivers a SPA-like user experience without the build complexity, bundle size, or client-state management overhead of React/Vue. The server stays authoritative — a particularly good fit for a server-authoritative gaming platform.

Static File Serving

In development (Docker Compose), Caddy serves /static/* files directly from /srv/static (mounted from web/static/). This means CSS/JS changes are immediately reflected without rebuilding the Go binary. In production, the same setup applies — Caddy serves static files while proxying API requests to Go.
