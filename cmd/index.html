📄 Entry Point

cmd/

The cmd/ directory contains the application's entry point following the standard Go project layout convention. Executable commands live here, each in their own subdirectory named after the binary they produce.

Directory Structure

PathPurpose
cmd/api/main.goApplication entry point — wires together config, logger, server, and graceful shutdown.

cmd/api/main.go

Package: main

This file is the only executable entry point for the PRJTPT API server. It is intentionally thin — its only job is to load configuration, construct the server, and manage the process lifecycle. All business logic lives in internal/.

Startup Sequence

  1. Call config.Load() to read all environment variables (exits on validation failure).
  2. Construct a zerolog.Logger via logger.New(cfg).
  3. Call server.New(cfg, log) which builds the full router, middleware stack, stores, and handlers — and opens the PostgreSQL connection pool.
  4. Register os.Signal listeners for SIGINT and SIGTERM.
  5. Start listening on the configured address (default :8080) in a goroutine.
  6. Block until a signal is received, then call http.Server.Shutdown with a 30-second context to drain in-flight requests before exiting.

Graceful Shutdown

The 30-second drain window allows WebSocket connections and long-lived SSE streams to finish before the process exits. Caddy's upstream health checks ensure no new requests are routed to a draining instance.

// Simplified representation of the startup logic
func main() {
    cfg  := config.Load()
    log  := logger.New(cfg)
    srv  := server.New(cfg, log)

    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)

    go srv.ListenAndServe()
    <-quit

    ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
    defer cancel()
    srv.Shutdown(ctx)
}

Convention: cmd/ packages should import from internal/ but never the other way around. This enforces that the entry-point is just a composition root with no business logic of its own.
