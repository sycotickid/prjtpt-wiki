📈 Database Schema

migrations/

PostgreSQL schema migrations managed by golang-migrate. Each migration is a numbered pair of .up.sql and .down.sql files. The migrator runs as a one-shot Docker Compose service before the API starts.

Migration Files

#FilesWhat It Creates
001000001_users.up.sql / .down.sqlCore users table with email/password auth, verification tokens, password reset, display name, avatar, status fields.
002000002_friends.up.sql / .down.sqlfriends table — directed friendship graph with status (pending/accepted/blocked).
003000003_games.up.sql / .down.sqlgames table — game catalogue (slug, title, description, player counts, screenshots).
004000004_channel_types.up.sql / .down.sqlchannel_types reference table seeded with: lobby, game, team, whisper.
005000005_messages.up.sql / .down.sqlmessages table — multi-channel chat (scoped by channel_type_id + channel_id).
006000006_lobbies.up.sql / .down.sqllobbies, lobby_members, and game_sessions tables.
007000007_push_subscriptions.up.sql / .down.sqlpush_subscriptions table — Web Push endpoint + p256dh + auth per user per device.
008000008_refresh_tokens.up.sql / .down.sqlrefresh_tokens table — server-side token rotation with family-based revocation.
009000009_teams.up.sql / .down.sqlteams and team_members tables (Phase 5 — pre-created but not yet used).
010000010_indexes.up.sql / .down.sqlPerformance and search indexes: trigram GIN on display_name, partial indexes on active users, pending friends, waiting lobbies, non-revoked tokens.
011000011_games_seed.up.sql / .down.sqlSeeds the games catalogue with initial game records (slugs, titles, descriptions, player counts).

Table Details

users

ColumnTypeNotes
idUUIDPrimary key, gen_random_uuid()
emailTEXTUnique (functional index on lower(email)), max 256 chars
password_hashTEXTbcrypt hash at cost 12
display_nameTEXTUnique, 3–32 chars, alphanumeric + _-. GIN trigram index for search.
avatar_urlTEXTS3 public URL, nullable
email_verifiedBOOLEANDefault false. Login blocked until true.
verification_tokenTEXTSHA-256 hash of the raw token sent by email. Cleared after use.
verification_token_expires_atTIMESTAMPTZ24-hour window
password_reset_tokenTEXTSHA-256 hash. Cleared after use.
password_reset_expires_atTIMESTAMPTZ1-hour window
statusTEXTEnum: online, offline, away, in_game, in_lobby, custom
custom_statusTEXTFree-text status when status = 'custom'
notifications_enabledBOOLEANDefault true. Gates Web Push delivery.
deleted_atTIMESTAMPTZSoft delete. Partial index on active users excludes deleted rows.
created_at / updated_atTIMESTAMPTZAuto-managed

friends

ColumnTypeNotes
idUUIDPrimary key
from_user_idUUIDFK → users(id)
to_user_idUUIDFK → users(id)
statusTEXTpending / accepted / blocked
created_atTIMESTAMPTZ

Constraints: no self-friendship (from_user_id != to_user_id), unique pair (UNIQUE (from_user_id, to_user_id) with a partial index excluding blocked).

games

ColumnTypeNotes

slugTEXTUnique, URL-safe identifier
titleTEXTDisplay name
subtitleTEXTOne-line tagline
descriptionTEXTFull description
categoryTEXTe.g. "Action", "Puzzle", "Strategy"
min_players / max_playersINTPlayer count constraints for lobbies
is_turn_basedBOOLEANAffects game loop implementation
screenshotsTEXT[]Array of image URLs
is_activeBOOLEANOnly active games appear in catalogue

ColumnTypeNotes

refresh_tokens

ColumnTypeNotes

revoked_atTIMESTAMPTZSet when rotated or explicitly revoked (logout)

Running Migrations

# Apply all pending migrations
make migrate-up

# Roll back the last migration
make migrate-down

# In Docker Compose, the migrator service runs automatically:
docker compose up

Tool: Migrations are run by the golang-migrate/migrate CLI, installed via make tools. The migrator Docker service (Dockerfile.migrator) runs the same CLI inside the container network so it can reach the Postgres service.
