PRJTPT Documentation

PRJTPT (Project Teapot) is a browser-based multiplayer gaming platform designed to operate behind corporate firewalls by presenting itself as productivity software. All game state is server-authoritative, computed in Go, and streamed to clients over HTTP/3 using WebTransport.

What is PRJTPT?

PRJTPT delivers real-time multiplayer games entirely through a web browser — no installs, no plugins. The platform uses a hypermedia-first architecture (Datastar) so pages feel like a modern SPA while remaining fully server-rendered. Game logic runs on the server and is broadcast to clients, eliminating cheating at the source.

The name is a reference to HTTP 418 I'm a teapot — a playful acknowledgment that the application masquerades as corporate tooling while actually being a gaming platform.

Architecture Overview

The application follows a layered architecture with strict separation between configuration, transport, business logic, and data access. The server is assembled via dependency injection in internal/server/server.go — stores are created, injected into handlers, and handlers are registered on the router.

internal/middleware/HTTP middleware: CORS, request logging, OWASP security headers, JWT auth, CSRF.

internal/server/Router wiring, dependency injection assembly, PostgreSQL connection pool.

web/Server-rendered HTML templates and static assets (CSS, JS, service worker, fonts).

Security Posture

OWASP Top 10 mitigations are implemented throughout. Parameterized queries prevent SQL injection; RS256 JWTs (15-min TTL) with server-side refresh-token rotation prevent session theft; double-submit CSRF cookies; OWASP security headers including strict CSP; bcrypt (cost ≥ 12); SHA-256 token hashing at rest.
